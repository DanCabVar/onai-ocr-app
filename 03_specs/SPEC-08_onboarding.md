# SPEC-08 — Onboarding usuario nuevo

## Tarea relacionada

- `T07` Onboarding usuario nuevo.

## Objetivo

Reducir fricción del primer uso: que un usuario nuevo entienda qué subir, cómo crear/inferir tipos y cómo obtener su primer resultado útil.

## Alcance preliminar

Incluye:

- pantalla o wizard inicial;
- CTA para crear primer tipo o inferir desde muestras;
- ejemplos de documentos;
- explicación breve de límites del plan;
- empty states útiles.

## Criterios de aceptación

- Usuario sin documentos sabe cuál es el siguiente paso.
- El onboarding no bloquea usuarios avanzados.
- Funciona en mobile/desktop básico.

## Verificación mínima

- Frontend build.
- Screenshot o prueba manual con usuario nuevo.
